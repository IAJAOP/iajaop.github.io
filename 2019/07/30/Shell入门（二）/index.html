网络相关命令

网络相关的命令按照如下几类进行讲述：

命令 作用 命令 作用
ping 网络是否通 telnet 端口是否通
netstat 端口是否被占用 ifconfig 主机的网卡情况
ssh，telnet 远程登录工具 ftp，sftp，scp 文件传输工具
  • 主机间网络连通性：ping IP
  • 测试另外一台服务器的端口是否开启监听，是否能访问：telnet IP 端口号
  • 检验本机各端口的网络连接情况：
    • 查看某端口被占用情况：netstat -apn|grep 端口号或者lsof –i:端口号
    • 查看某个进程占用了哪些端口：netstat -apn|grep pid
  • 查看网卡和设置网卡：ifconfig
    • 用来查看和配置网络设备
    • 当网络环境发生改变时可通过此命令对网络进行相应的配置

命令 作用 命令 作用

文件传输

命令 作用 命令 作用
scp 全拼secure copy，用于不同主机之间复制文件 ftp 通过ftp协议进行传输
sftp 通过ssh进行传输
  • scp文件传输：scp [参数] [源路径] [目标路径]
    • 文件复制到远程服务器：scp local_file remote_username@remote_ip:remote_folder
    • 目录复制到远程服务器：scp -r local_folder remote_username@remote_ip:remote_folder
    • 远程服务器到本地：scp remote_username@remote_ip:remote_file local_folder
  • ftp文件传输：
    • 登录到远程服务器：ftp IP （输入用户名&密码）
    • 常用两种传输模式：
      • ascii：将文件传输设置为网络ASCII，默认使用ascii传输
      • binary：将文件传输设置为二进制传输，需要使用二进制传输的有：iso文件、可执行文件、压缩文件、图片等等
  • sftp传输：
    sftp -Oport=端口号 username@passwd也有ascii传输和bin传输

进程管理

命令 作用 命令 作用
ps，top 查看进行的运行状态 kill 杀死进程

ps：提供系统过去信息的一次性快照；
top：反应的是系统进程动态信息，默认10s更新一次。

eg：ps aux：

  • USER：进程拥有者,示例中是root
  • PID：进程ID，用户ID为UID，父进程ID为PPID
  • %CPU：占用的CPU使用率
  • %MEM：占用的物理内存百分比
  • VSZ：占用的虚拟内存量
  • RSS：占用的固定的内存量
  • TTY：终端的次要装置号码（minor device number of tty
  • STAT：该进程的状态，有下一个板块的几个状态，D，R，S，T，Z是ps指令标识进程的5种状态码
  • TIME：进程已消耗的CPU时间
  • CMD：启动进程的命令

eg：ps -ef：

  • UID：用户ID、但输出的是用户名
  • PID：进程的ID
  • PPID：父进程ID
  • C：进程占用CPU的百分比
  • STIME：进程启动到现在的时间
  • TTY：该进程在那个终端上运行，若与终端无关，则显示‘?’；若为‘pts/0’等，则表示由网络连接主机进程
  • CMD：命令的名称和参数

常用的命令：

  • ps -ef|grep 进程名查看进程的运行状态、以及父子进程关系
  • kill -9 pid杀掉一个进程pid

Shell的基础知识

Shell变量类型和符号

  1. 变量的定义
  • 命名只能使用英文字母，数字和下划线，首个字符不能以数字开头；
  • 中间不能有空格，可以使用下划线（_）；
  • 不能使用标点符号；
  • 不能使用bash里的关键字（可用help命令查看保留关键字）
    有效的命令名实例如下：RUNOOB、LD_LIBRARY_PATH、_var/var2。
  1. 变量声明和使用
  • 变量是弱类型（不用声明类型）；
  • 变量声明及赋值格式：变量=值（等号两边不能有空格）；
  • linux默认变量都是做字符串处理；
  • 清除变量：unset 变量名。

  1. 变量的赋值
    name=my与name=“my”等效
    name=“my bird”
    注意：1）变量类型不需要申明 ;2）赋值内容无空格，可不加“”。

  2. 变量的引用
    $变量名或者${变量名}（推荐以这种方式访问）。

  3. 变量的类型
    Shell变量，可以保存如路径名、文件名或者一个数字，主要存在一下几种变量：

  • 本地变量：局部变量只在创建它们的shell中使用，可以在shell程序内任意使用和修改它们；
  • 环境变量：可以在创建他们的shell以及派生出来的任意子程序中使用；
  • 内部变量：有系统提供的，与环境变量不同，但是用户不能修改它们。
    其中：
    • 设置环境变量：环境变量的命名需要大写，且需要用export进行定义export DPS_PATH=/cbs/card
    • 查看环境变量：
      • env查看全部环境变量
      • echo $环境变量名显示一个环境变量
    • 清除环境变量: unset 环境变量名（eg：unset DPS_PATH）
  1. 变量定义中的字符串
    • 单引号：
      被引起来的字符全部做普通字符，即全部原样输出：echo 'my $SHELL'，这里$SHELL不会被转换。
    • 双引号：
      引号内的内容，除了$、转义符\，倒引号（`），这三个保留特殊的功能，其他均作普通字符。
      典型的一个例子：“‘$var’”单引号和双引号典型的是，如果最外面是双引号，那么里面的$var进行转换，如果最外面是单引号''，里面不做转换，直接输出。

环境变量

  • HOME：代表使用者的主文件夹，也就是用户登录的默认目录，或者cd 默认回到的目录。export HOME=指定目录,可以人为的改变。
  • PATH：就是可执行文件搜寻的路径啦～目录与目录中间以冒号（:）分隔， 由于文件的搜寻是依序由PATH的变量内的目录来查询，所以，目录的顺序也很重要。
    eg：执行某个可执行程序run.sh
    ./run.sh当前目录下的是这个
    export PATH=当前路径:$PATH，则可以通过run.sh执行。

存放环境变量的文件：

  • /etc/profile该文件的作用是当用户登录时获取系统的环境变量，只获取一次；
  • /etc/bashrc当执行完/etc/profile文件后，用户想打开bash Shell就会读取该文件。如果想每次打开bash Shell后都执行某些操作，可以在该文件中设置；
  • ~/.bash_profile每个用户都可使用该文件输入专用于自己使用的shell信息。当用户登录时，该文件仅仅执行一次，默认情况下，它设置一些环境变量，执行用户的.bashrc文件；
  • ~/.bashrc该文件包含专用于单个人的bash Shell的bash信息，当登录时以及每次打开一个新的Shell时，该文件被读取。

内部变量

内部变量是linux系统所提供的一种特殊类型的变量，这种变量只能在程序中做判断，不能修改。
部分内部变量如下：

  • $?：最后命令的完成码或者在shell程序内部执行的shell程序（返回值）
  • $0：shell程序的名称
  • $$：本程序的PID

输入输出重定向

重定向是指用户改变程序默认的输入和输出。
linux系统中一切皆文件，一般情况下，每个Linux命令运行时都会打开三个文件：

  • 标准输入文件(stdin)：stdin的文件描述符为0，程序默认从stdin读取数据。
  • 标准输出文件(stdout)：stdout的文件描述符为1，程序默认向stdout输出数据。
  • 标准错误文件(stderr)：stderr的文件描述符为2，程序会向stderr流中写入错误信息。

输入重定向

符号 作用
< 输入重定向运算符
<< 告诉Shell,当前标准输入来自命令行一对分割号中间的内容

输出重定向

符号 作用
> 输出重定向运算符，会自动建立或者覆盖
>> 表示第二次的内容加在原来的文件上

Shell书写规范

书写shell的思路大致如下：

  1. 整理需求，写出设计方案
  2. 用命令行的方式来调试所要用到的命令
  3. 书写shell脚本，不确定的地方调通了再写
  4. 书写注释
  5. 整体测试

其中第3步的书写shell脚本具体的是：

  • 开头指定脚本解释器#!/bin/bash或者#!/bin/sh
  • 开头加版权信息
  • 一些知识点或者业务信息用#注释
  • 缩进
  • 变量定义要有含义

注意：

  1. 一般shell文件以.sh文件后缀结尾，没有也可以执行
  2. 常见的执行方式有：
    • ./文件名
      chmod +x run.sh
      ./run.sh
    • sh 文件名
      sh run.sh
  3. 对于linux和Unix之间移植的，需要调试。
