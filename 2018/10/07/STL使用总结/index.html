STL容器使用总结

Posted on 2018-10-07 | In C++ | 本文总阅读量 次
Words count in article: 2.1k字 | Reading time ≈ 8分钟

STL容器

用来管理某类对象的集合。STL中可分为三类容器：序列式容器（Sequence）、关联式容器（Associative）、无序容器（Unordered）。不同类别容器彼此之间完全不同，有着不同的实现，彼此没有派生关系。

  • 序列式容器通常被实现为array或linked list；
  • 关联式容器通常被实现为binary tree；
  • 无序容器通常被实现为hash table。

容器的共通能力

三个最核心的能力是：

  • 所有容器提供的都是”value语义“而非”reference语义“；
  • 元素在容器内有其特定的顺序，每种容器都会提供若干“返回迭代器”的操作函数，这鞋迭代器可用来遍历各个元素；
  • 一般而言，各项操作并非绝对安全。
    基本共通的操作：
  • ContType c，default构造函数，建立一个未含任何元素的空容器（array将获得默认元素）；
  • ContType c(c2)，copy构造函数，建立一个新容器为c2的拷贝（复制所有元素）；
  • ContType c = c2，copy构造函数，建立一个新容器为c2的拷贝（复制所有元素）；
  • ContType c(rv)，move构造函数，建立一个新容器，取rvalue rv的内容为初始值（C++11后不适用于array）；
  • ContType c = rv，move构造函数，建立一个新容器，取rvalue rv的内容为初始值（C++11后不适用于array）；
  • ContType c(beg, end)，建立一个容器以[beg,end)区间内的所有元素拷贝为初始值（不适用于array);
  • ContType c(initlist)，建立一个容器，并以初值列initlist的拷贝为初值（C++11后不适用于array）；
  • ContType c = initlist，建立一个容器，并以初值列initlist的拷贝为初值（C++11后不适用于array）；
  • c.~ContType()，删除所有元素，释放内存（如果可能的话）；
  • c.empty()，判断容器是否为空（比size() == 0更快）；
  • c.size()，返回容器中的元素量（不适用于forward_list)；
  • c.max_size()，返回元素的最大可能数量；
  • c1 == c2，判断是否c1等于c2；
  • c1 != c2，判断是否c1不等于c2；
  • c1 < c2，判断是否c1小于c2（不适用于unordered容器）；
  • c1 > c2，判断是否c1大于c2（不适用于unordered容器）；
  • c1 <= c2，判断是否c1小于等于c2（不适用于unordered容器）；
  • c1 >= c2，判断是否c1大于等于c2（不适用于unordered容器）；
  • c = c2，将c2的所有元素赋值给c；
  • c = rv，将rvalue rv以move assign方式赋予c（C++11后不适用于array）；
  • c = initlist，将初值列initlist中的所有元素赋值给c（C++11后不适用于array）；
  • c1.swap(c2)，置换c1和c2的数据；
  • swap(c1, c2)，同上，是个全局函数；
  • c.begin()，返回一个iterator，指向第一个元素；
  • c.end()，返回一个iterator，指向最末尾元素的下一位置；
  • c.cbegin()，返回一个const iterator,指向第一个元素（始自C++11）；
  • c.cend()，返回一个const iterator，指向最末尾元素的下一位置（始自C++11）；
  • c.clear()，移除所有元素，令容器为空（不适用于array）。

C++标准库中的容器

序列式容器

是一种有序集合。
《C++标准库》第二版中有array、vector、deque、list、forward_list。
《STL源码分析》中有vector、list、deque、stack、queue、heap、priority_queue、slist。

array

头文件#incude <array>
array是TR1新引入的，进入C++标准库的时间较短，而切有一些特殊性，与其他STL容器不共同。
一个array对象是在某个固定大小的array（可称为static array或C array）内管理元素。因此不可以改变元素个数，只能改变元素值。必须在建立时就指明其大小。

vector

头文件#include <vector>
vector讲其元素置于一个dynamic array中管理，允许随机访问，可理解为一个动态数组，存储空间是连续的，从尾部添加或删除元素都很快。
Vector扩容时会重新分配内存，一旦内存重新分配，vector元素相关的所有reference、pointer、iterator都会失效。
Vector不能使用reserve()缩减容量，利用swap函数能巧妙缩减容量。

deque

头文件#include <deque>
deque是一个dynamic array，可以向两端发展，从尾部或头部安插元素都十分迅速，存储空间是分块连续的。deque通常实现为一组独立区块，第一区块朝某方向扩展，最末区块朝另一方向扩展，所以迭代器需要在不同区块间挑战。
Deque不支持对容量和内存重新分配时机的控制，且不必在内存重新分配时复制所有元素。除了头尾两端，在任何地点安插或删除元素都将导致指向deque元素的任何pointer、reference和iterator失效。

list

头文件#include <list>
list由双向链表实现，不提供随机访问。优势是在任何位置上执行安插和删除动作都十分迅速。
List对象自带两个pointer，用老指向第一个和最后一个元素。
安插和删除不会造成各个pointer、reference和iterator失效。

forward_list

头文件#include <forward_list>
forward_list由单向链表实现，可节省内存，不提供随机访问，只提供向前迭代器，而不是双向迭代器。因此它不支持返向迭代器，意味着没有reverse_iterator之类的类型以及rbegin()、rend()之类的成员函数。

关联式容器

是一种已排序集合，元素位置取决于value（或key，如果元素是个key/value pair）和给定的某个排序准则。
《C++标准库》第二版中有set、multiset、map、multimap。
《STL源码分析》中有set、multiset、map、multimap、hash_set、hash_map、hash_multiset、hash_multimap。

set

头文件#include <set>
set是“以节点(node)为基础”的容器。
元素依据其value自动排序，每个元素只能出现一次，不允许重复，自动排序造成set的一个重要限制：不能直接改变元素值，这回打乱原本正确的排序。因此，要改变元素值，必须先删除旧元素，再插入新元素。
set事实上通常以红黑树实现，红黑树在改变元素数量和搜寻方面都很出色，红黑树保证节点安插时最多只会做两个重新链接动作，而且到达某一元素的最长路径深度至多只是最短路径深度的两倍。

multiset

头文件#include <multiset>
和set的唯一差别是，元素可以重复。

map

头文件#include <map>
每个元素都是key/value pair，其中key是排序准则的基准，每个key只能出现一次，不允许重复，通常以平衡二叉树来实现。
key和value都必须是可复制的（copyable）或可搬移的（moveable），对指定的排序准则而言，key必须是可比较的（comparable）。

multimap

头文件#include <multimap>
和map的唯一差别是，元素可以重复，也就是multimap允许其元素拥有相同的key。Multimap可被当作字典使用。

无序容器

是一种无序集合，其内每个元素的位置无关紧要，唯一重要的是某个特定元素是否位于此集合内。通常以hash table实现，内部结构是一个”由linked list组成“的array。通过某个hash函数的运算，确定元素落于这个array的位置。
无序容器的主要优点是，查找速度可能快过关联式容器。
《C++标准库》第二版有4种无序容器，无序容器由TR1引入。

unordered_set

头文件#include <unordered_set>
是无序元素的集合，每个元素只可以出现一次。

unordered_multiset

头文件#include <unordered_set>
和unordered_set唯一差别是它允许元素重复出现。

unordered_map

头文件#include <unordered_map>
元素都是key/value pair，每个key只可以出现一次。

unordered_multimap

头文件#include <unordered_map>
和unordered_map的唯一区别是允许key重复。

其他容器

string

头文件include <string>
可以把string当作一种STL容器，很类似vector，但其元素都是字符。

寻常的C-Style Array

由C/C++语言内核提供，是个type而不是个class。

容器适配

容器适配是STL的一部分，是STL内核的外围。

stack

头文件#include <stack>
对元素采取先进后出管理策略。

queue

头文件#include <queue>
对元素采用先进先出管理策略。

priority_queue

头文件#include <priority_queue>
其内元素拥有各种优先权。
