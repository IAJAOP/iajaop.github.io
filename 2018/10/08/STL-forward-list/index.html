STL-forward_list

Posted on 2018-10-08 | In C++ | 本文总阅读量 次
Words count in article: 1.1k字 | Reading time ≈ 4分钟

forwardlist操作

构造函数和析构函数

  • forward_list<Elem> c，default构造函数，产生一个空forward_list，没有任何元素；
  • forward_list<Elem> c(c2)，copy构造函数，建立c2的同型forward_list并成为c2的一份拷贝；
  • forward_list<Elem> c = c2
  • forward_list<Elem> c(rv)，move构造函数，取rvalue rv内容构造一个新的forward_list（始自C++11）；
  • forward_list<Elem> c = rv
  • forward_list<Elem> c(n)，利用元素的default构造函数生成一个大小为n的forward_list；
  • forward_list<Elem> c(n, elem)
  • forward_list<Elem> c(beg, end)
  • forward_list<Elem> c(initlist)
  • forward_list<Elem> c = initlist
  • c.~forward_list()，销毁所有元素，释放内存。

非更易型操作

forward_list不提供成员函数size()。

  • c.empty()
  • c.max_size()，返回元素个数之最大可能量；
  • c1 == c2
  • c1 != c2
  • c1 < c2
  • c1 > c2
  • c1 <= c2
  • c1 >= c2

赋值

  • c = c2
  • c = rv，将rvalue rv的所有元素以move assign方式给予c（始自C++11）；
  • c = initlist
  • c.assign(n, elem)， 复制n个elem，赋值c；
  • c.assign(beg, end)
  • c.assign(initlist)
  • c1.swap(c2)
  • swap(c1, c2)

元素访问

访问forward_list的所有元素，必须使用rang-base for循环，特定操作函数或迭代器。
forward_list不支持随机访问。

  • c.front()，返回第一元素，不检查；

迭代器相关函数

只提供向前迭代器，而不是双向迭代器。因此它不支持返向迭代器，意味着没有reverse_iterator之类的类型以及rbegin()、rend()之类的成员函数。
带const的都始自C++11。

  • c.begin()，返回一个random-access iterator指向第一元素；
  • c.end()，返回一个random-access iterator指向最末元素的下一位置；
  • c.cbegin()，返回一个const random-access iterator指向第一元素；
  • c.cend()，返回一个const random-access iterator指向最末元素的下一位置；
  • c.before_begin()，返回一个forward iterator指向第一元素的前一位置；
  • c.cbefore_begin()，返回一个const forward iterator指向第一元素的前一位置。

安插和移除元素

用insert_after()取代insert()，将新元素安插于第一实参所表示的元素之后。

  • c.push_front(elem)，在头部插入elem的一个拷贝；
  • c.pop_front()，移除第一元素，但不返回；
  • c.insert_after(pos, elem)，在iterator位置pos之后插入1个elem拷贝，并返回新元素的位置；
  • c.insert_after(pos, n, elem)，在iterator位置pos之后插入n个elem拷贝，并返回第一个新元素的位置（n == 0，返回pos）；
  • c.insert_after(pos, beg, end)
  • c.insert_after(pos, initlist)
  • c.emplace_after(pos, args)，在iterator位置pos之后插入1个arg为初值的元素，并返回新元素的位置（始自C++11），相对于insert_after函数执行构造函数、拷贝构造函数，emplace只执行构造函数；
  • c.emplace_front(args，附加一个以args为初值的元素于头部，不返回；
  • c.erase_after(pos)，移除iterator位置pos之后的元素，返回下一元素位置；
  • c.erase_after(beg, end)
  • c.remove(val)，移除所有其值为val的元素；
  • c.remove_if(op)，移除所有“造成op(elem)结果为true“的元素；
  • c.resize(n)，将元素数量改为n，若size()变大，多出来的新元素都需以default构造函数完成初始化；
  • c.resize(n, elem)，将元素数量改为n，若size()变大，多出来的新元素都是elem的拷贝；
  • c.clear()，移除元素，将容器清空。

其他改变元素的操作

  • c.unique()，若存在若干相邻而数值相同的元素，则移除重复元素，只留一个；
  • c.unique(op)，若存在若干相邻元素都使op()的结果为true，则移除重复元素，只留一个；
  • c.splice_after(pos, c2)，将c2内的所有元素转移（move）到c之内、迭代器pos之后
  • c.splice_after(pos, c2, c2pos)
  • c.splice_after(pos, c2, c2beg, c2end)
  • c.sort()，以operator < 为准则对所有元素排序；
  • c.sort(op)以op()准则对所有元素排序；
  • c.merge(c2)，假设c和c2容器都包含op()准则下的已排序元素，将c2的全部元素转移到c，并到证合并后的forward_list仍为已排序；
  • c.merge(c2, op)，假设c和c2容器都包含已排序元素，将c2的全部元素转移到c，并保证合并后的forward_list在op()准则下仍为已排序；
  • c.reverse()，将所有元素反序。

异常处理

forward_list对于异常安全性提供了最佳支持，所有操作都是不成功便成仁：要么成功，要么无效。
