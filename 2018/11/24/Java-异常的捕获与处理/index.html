throws

在进行方法定义的时候，如果告诉调用者可能产生哪些异常，就用使用throws进行声明。也可以说如果该方法出现问题后，不希望进行处理，就使用throws抛出。
如果调用了throws声明的方法，在使用时就必须明确的使用try...catch进行异常捕获，因为该方法有可能产生异常。

主方法本身也是一个方法，所以主方法可以使用throws进行异常的抛出，这个时候如果产生异常就会交给JVM来处理。

面对未知的程序类，如果要进行异常处理，就必须明确的知道有多少种异常。

throw

throw是直接编写在语句当中的，表示人为的进行异常的抛出。不希望异常类对象由系统产生，只是希望用户来控制异常类实例化对象的产生，就可以使用throw产生。

throws和throw区别：
| - | 区别 |
|——–|————————————————————————————|
| throws | 主要在方法声明上使用，明确的告诉用户该方法可能产生异常，同时该方法可能不处理此异常 |
| throw | 用于方法内部主要表示进行手工的异常抛出 |
