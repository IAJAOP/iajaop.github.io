Java-多线程

如果想实现多线程，那么必须有一个前提：有一个线程的执行主类。有两类途径来实现一个多线程的主类：

  • 继承一个Thread类；
  • 实现Runnable、Callable接口。
    在多线程的处理上使用的是代理设计模式。
    实际上程序主方法本身就是一个线程，所有的线程都是通过主线程创建并启动的。

Thread类

java.lang.Thread是一个线程操作的核心类。如果想要定义一个线程的主类，那么最简单的方法就是直接继承Thread类，而后覆写该类中的run()方法（相当于线程的主方法）。

多线程常用操作方法

多线程的主要的操作方法都在Thread类里面。

  1. 线程的命名和取得

    • public Thread(Runnable target, String name)，创建线程对象时设置线程名字；
    • public final void setName(String name)，设置线程名字；
    • public final String getName()，取得线程名字；

  2. 取得当前线程对象

    • public static Thread currentThread()

  3. 线程休眠，让线程暂缓执行，到预计时间时再执行

    • public static void sleep(long millis) throws InterruptedException，使用毫秒作为时间单位；

  4. 线程优先级，优先级越高，可能越有限执行

    • 设置优先级，public final void serPriority(int newPriority)；
    • 取得优先级，public final int getPriority()；
      对于优先级的设置内容可通过Thread类中的常量来设置：
    • 最高优先级：public static final int MAX_PRIORITY，值为10；
    • 中等优先级：public static final int NORM_PRIORITY，值为5；
    • 最低优先级：public static final int MIN_PRIORITY，值为1；

线程的同步与死锁

上述代码不同步，数据不安全，不同步的唯一好处就是处理速度快，多个线程并发执行。
同步的本质指多个操作在同一时间段内只能有一个线程进行，其他线程要等待此线程完成之后才可以继续执行。

wait和notify方法

Java多线程开发中，常用到wait()和notify()方法来实现线程间的协作，简单的说步骤如下：

  1. A线程取得锁，执行wait()，释放锁;
  2. B线程取得锁，完成业务后执行notify()，再释放锁;
  3. B线程释放锁之后，A线程取得锁，继续执行wait()之后的代码；

sleep、yield和join方法

  • sleep()使当前线程进入阻塞状态，在指定时间内不会执行。
  • yield()暂停当前正在执行的线程对象。yield()只是使当前线程重新回到可执行状态，所以执行yield()的线程有可能在进入到可执行状态后马上又被执行。yield()只能使同优先级或更高优先级的线程有执行的机会。
  • join()等待该线程终止。等待调用join()方法的线程结束，再继续执行。
